# 流程表单增强
当前的项目需要合并featuer-run分支后，就可以正确在本地运行，目前的项目中，流程的功能实现都已经完整，但是有一些需要增强的地方
## 表单增强功能点
1. 合并feature-run分支到当前项目，确保项目可以正确运行后，再进行后续的操作，防止将一个正确的项目修改错误。也确保修改所有的修改都有一个正确的基础。
2. 目前的表单设计有一个十分不好的问题，就是页面无法布局，每个元素都是独占一行，这就导致表单无法设计出美观合理的布局，需要利用网格布局的思想，在将元素推放到表单中时，可以设置其布局的位置，而设置的形式和当前项目设置元素的功能是一致的。
3. 目前这个表单已经内置了很多元素，为了增强功能，需要加入一个AI-UI组件，这个组件可以是一个组件系列，包括了AI大模型的大多数功能，这个功能是应用场景上的，比如聊天机器人，智能客服，智能推荐，图片AI剪裁，视频AI处理等，这些功能都可以通过AI-UI组件来实现。最后达到的效果就是在绘制表单的时候，就可以拖入这些AI元素，从而实现一个包含AI功能的表单。
4. 还有一个就是在设计表单的时候，加入一个AI绘制表单的功能，通过用户描述，来进行表单的绘制。这个功能的实现，需要利用AI大模型的能力，通过用户描述，来生成对应的表单元素，并且设置其布局。

## 开发注意事项
1. 前后端的端口不能变，如果发现已经启动的，就先关闭q，再启动。
2. 前端的代码修改后，需要重新编译，才能生效。
3. 后端的代码修改后，需要重新启动，才能生效。
4. 一定要有全局意识，在修改代码的时候，要考虑到整个项目的影响，不要只修改当前的模块，而忘记了其他模块的影响。而且当前项目本身就是正确的可以运行的，千万不要改错项目。
5. 切记不能伪造，模拟数据，要真实的实现功能。
