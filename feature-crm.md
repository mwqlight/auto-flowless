# 客户关系管理系统（CRM）
在当前项目创建一个子模块auto-flowless-crm-starter,并集成到web中使用，注意不是单独模块而是子模块，业务功能使用的路径在建立客户关系管理菜单，子菜单为客户线索挖掘，画像分析、信贷风险评估、营销活动管理、业务跟进与绩效统计等。用于实现客户关系管理系统的功能。
## 系统核心目标
1. 构建一个覆盖对公客户全生命周期的精细化管理系统，支持客户线索挖掘、画像分析、信贷风险评估、营销活动管理、业务跟进与绩效统计，最终提升银行对公业务的获客效率和客户留存率。

## 核心模块设计
1. 客户信息管理
功能要点：
客户基础信息（名称、行业、规模、控股股东等）的CRUD操作。
客户分级管理（如按资产规模分为A/B/C类），支持标签化标记（JSON格式存储动态标签，如 ["VIP","长期合作"]）。
关联联系人、合作历史、业务往来记录。
技术实现：
后端：Spring Boot + MyBatis Plus，设计CrmCustomer表，包含tags字段（JSON类型）。
前端：Vue3 + Element Plus，提供表格展示、筛选（按行业/等级）和标签编辑组件。
2. 线索与商机管理
功能要点：
线索来源管理（如公开数据抓取、银企沙龙活动录入）。
线索分配至客户经理，跟踪跟进状态（未联系、已接触、转化中、已成交）。
商机阶段管理（意向阶段、方案沟通、合同审批、成交）。
技术实现：
后端：设计CrmClue（线索）和CrmOpportunity（商机）表，通过状态枚举实现流程控制。
前端：看板视图展示各阶段商机，支持拖拽更新状态。
3. 客户画像与风险分析
功能要点：
自动化客户画像：整合企业财务数据（营收、负债率）、舆情信息（诉讼、处罚记录），生成风险评级（低/中/高）。
信贷分析报告：基于行业政策（如光伏产业）和企业数据，自动生成信贷建议。
技术实现：
后端：集成规则引擎（如Drools），根据财务指标（资产负债率、现金流）自动计算风险等级。
前端：ECharts图表可视化客户风险分布、行业趋势。
4. 营销活动管理（银企沙龙）
功能要点：
活动全流程管理：方案策划（PDCA模型）、企业邀约、现场执行、后期跟进。
“3-7-30”跟进机制：活动后3天内发资料、7天内回访、30天复盘。
技术实现：
后端：设计CrmCampaign表，关联参与企业、活动反馈字段。
前端：工作流表单配置活动节点，自动触发跟进任务。
5. 业务执行与跟踪
功能要点：
客户经理工作台：每日跟进任务提醒、客户动态更新。
合同与回款管理：记录合同金额、回款计划、逾期预警。
技术实现：
后端：Spring Cloud Stream实现消息队列，自动推送任务提醒。
前端：日历视图展示回款计划，颜色标记逾期状态。
6. 数据权限与多租户隔离
功能要点：
RBAC权限控制：不同角色（客户经理、管理员）仅可访问授权数据。
数据自动回收：超期未跟进的客户自动回收到公海池。
