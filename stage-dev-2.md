# 第二阶段开发要求
 该项目的第一阶段开发要求和环境说明在README.md文件中。需要结合第一阶段的开发要求和环境说明，完成第二阶段的开发。这是一个复杂的分阶段的项目开发。
 ## 核心开发要求
  1. 项目已经有了前后端的启动脚本，需要利用这个脚本正确启动前后端，当前这涉及到以下的基本工作，需要完成。
    - 确保前后端的启动脚本可以在本地环境正常运行。
    - 确保按照项目sql要求进行数据库的初始化工作。
    - 确保前端的node使用nvm切换到20.10。
    - 确保启动前后端没有错误和警告信息。
  2. 第一阶段的开发中有一个AI实验室的功能，需要你继续完成并且完成前后端的联调工作，确保前端调用的接口都是真实可用的，当然涉及到新的数据库表的创建，这个创建的逻辑就是前端页面的基本增删改查的操作。
  3. 将第一阶段的开发要求和环境说明中的内容，从README.md文件分离出来，文件的命名规则和stage-dev-阶段序号.md文件相同。然后重新完善README.md文件。
  4. 将前后端的项目文件名称分别改为auto-flowless-frontend和auto-flowless-backend。
