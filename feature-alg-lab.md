# 算法实验室
需要在当前项目添加一个算法实验室的菜单，然后其子菜单都是各个算法，然后点击各个算法主要有两个页面功能，一个是动画演示，一个是代码演示。因为同一个算法有不同的实现，因此可以有下拉框来选择不同的实现。而演示就是根据选择的实现来展示动画或代码，还需要有时间复杂度和空间复杂度的展示。
## 具体功能
1. 在auto-flowless-backend添加一个算法实验室的模块，auto-flowless-alg-lab-starter,需要集成到web中进行使用。
2. 需要利用sql进行算法实验室的菜单初始化，数据库密码和用户都是root,注意不能人工输入，不然就没有意义。
3. 要注意菜单一定要有图标，图标的内容可以自行选择。
4. 对于有多个算法的实现，需要有下拉框实现，页面的核心元素，就是动画演示和代码演示的区域，以及时间复杂度和空间复杂度的展示区域。
5. 算法种类主要有常用的算法,从数据结构去分大类，比如线性数据结构、树结构、图结构等。然后是线性数据结果下的各种算法，比如栈、队列、链表等。数结构下的各种算法，比如二叉树、二叉搜索树、堆等。图结构下的各种算法，比如深度优先搜索、广度优先搜索、最短路径等。再有就是通用算法，比如排序算法、搜索算法，动态规划，回溯算法，组合算法，贪心算法等，还有就是特殊算法，并查集，线段树，前缀树等，还有大模型算法，负载均衡算法，分布式算法，限流算法等，期望在算法实验室能够将能想到的所有算法都纳入进来。
6. 还需要添加一个扩展功能，利用AI进行来添加算法到算法实验室的功能，用户可以输入算法的名称，然后AI会根据名称来添加对应的算法到算法实验室中。然后达到和其他算法一样的功能。

## 注意事项
1. 要有全局意识，这个项目是能够在本地正确运行的，切不能将原有的功能修改错误。
2. 要注意代码的质量，不能有错误和警告信息。
3. 要注意AI添加算法的功能，要能够根据用户输入的名称来添加对应的算法到算法实验室中。而且是需要能够接入大模型管理大模型以后，才能有这样的能力。
4. 对应算法的分类，就目前分为：常用算法、数据结构算法、图算法、特殊算法、大模型算法、负载均衡算法、分布式算法、限流算法等。然后每个分类下，利用下拉框来选择具体的算法。对于后续扩展的算法，就列为扩展算法分类。
