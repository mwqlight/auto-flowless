# 第一阶段开发要求
## 项目的基本运行环境
1. 为了防止端口使用爆炸，需要你写一个前后端的shell脚本，需要按照项目要求的端口进行绑定，其次需要每次启动之前进行端口检查，如果存在启动的应用，就先关闭，然后再启动。力求保障前后的服务端口一直不变。
2. 数据库使用本地的localhost，端口为3306，数据库名称为auto_flowless，用户名和密码都是root。当然这些在你了解项目后需要进行修改。
3. Redis也是本地的localhost，端口为6379，数据库为1，密码为空。当然这些在你了解项目后需要进行修改。
4. 启动前端的时候需要特别注意，需要使用nvm切换到node版本为20.10以后才可以进行操作，这些需要内化到你的shell脚本中。
5. 这是一个前端是vue3,后端是springboot的项目，你需要在理解项目的基础上，根据项目的要求，以及项目的sql，进行本地数据库的初始化。初始化的sql文件在项目的sql目录下。

## 第一阶段的改造
在前面的提到的项目运行环境中，是你需要改造的一些基本原则，你需要根据这些原则，来进行项目的改造。具体的改造点如下：鉴于这是一个长期的复杂的任务，我们需要分阶段来进行改造：
1. 按照要求创建前后端的启动脚本。
2. 按照要求进行数据库的初始化，然后进行前后端的正常启动。
3. 在UI层面把logo的图片已经文字都修改为auto-flowless。
4. 添加一个AI实验室的菜单，其子菜单为AI模型管理，AI任务管理，AI模型训练，AI任务监控，AI模型评估,AI应用。在AI应用的界面分别提供RAG,KAG,AGENT,EMBEDDING，REACT的功能应用，通过TAB页面来区别，每个功能应用都有自己的界面，分别提供功能的配置和使用。这是一个涉及前后端的较为复杂的功能，首页菜单的功能，可以参照之前项目已经实现的功能，进行添加，注意添加的数据库ID必须要承接之前最大的那个ID序号，其次关于各个业务功能的页面UI,需要根据当前最成熟的业务模型来构建，具体流程就是核心参数需要能配置，效果需要能展示，页面风格需要和整体项目保持一致。
5. 添加gitignore文件，忽略一些不必要的文件，比如node_modules,target等。
