# 项目初始化准备开发任务
这是一个后端是java,使用springboot框架,数据库是mysql,redis,前端是vue3,使用element-plus组件库的前后端的项目，主要业务是以流程为基础的工作流管理系统。
这个项目是从开源项目flyflow中fork出来的，flyflow是一个基于流程的工作流管理系统，主要业务是流程的定义、流程的执行、流程的监控等。为了后续的整体开发任务，需要先做一些初始化的开发任务，是比较简单的，主要目的就是让这个项目能够本地运行起来，期间会有很多问题，需要解决。
## 核心初始化开发任务
1. 数据库的初始化，根据项目的sql脚本，初始化数据库，数据库的的本地连接信息需要修改为项目的本地连接信息。数据库的用户名和密码都是root,数据库的连接端口是3306，redis的连接端口是6379，没有密码，需要注意的是，你启动运行时，你需要将其修改到本地的信息，否则就会出错。
2. 前后端的启动脚本，确保可以在本地环境正常运行，如果有错误，需要进行修复。
3. 前端的启动需要将node使用nvm切换到20.10。
4. 确保启动前后端没有错误和警告信息。
5. 运行起来的标准就是能够访问前端页面，并且可以正常登陆，用sql中的管理员账户进行登陆，能够看到全部的菜单，以及各个功能没有问题，因为这个项目本身就是正常的可以运行的开源项目。
