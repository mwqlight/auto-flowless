# AI大模型管理功能
需要单独开发一个功能模块，用于管理AI大模型的相关操作，包括模型的上传、下载、训练、评估等。这个功能是比较复杂的，需要考虑到模型的存储、训练、评估等方面。所以需要尽可能的利用已有的开源框架来实现，但是这个项目的开发语言是java的，对于大模型的官方开发语言python来说不是很友好，所以需要在java项目中调用python的代码来实现大模型的相关操作。
## 功能模块的设计
1. 因为大模型的jdk比较新，所以需要升级jdk到jdk17,然后创建一个auto-flowless-llm-manager-starter模块，集成到web模块中来使用。
2. 为了能够在java项目中调用python的代码，需要使用python的jython库。jython是一个python的解释器，它可以在java项目中直接调用python的代码。所以需要在auto-flowless-llm-manager-starter模块中引入jython库的依赖。
3. 因为需要测试大模型，因此需要引入spring-ai-alibaba来进行大模型的调用测试。
4. 那pyhton的代码在哪里呢？需要建立一个auto-flowless-llm-manager-python模块，用于存放python的代码。
5. 这个独立的功能重要在于是利用已经有的开源框架来实现大模型的上传，下载，训练和评估，而不是从0到1开始开发，所以重要的功能是集成已有的开源框架，这很重要。其次就是菜单页面的设计，需要单独一个大模型管理的菜单下，包含模型上传、下载、训练、评估等子菜单。
6. 而且这个功能菜单，在每个功能页，需要有详细的说明，包括参数的配置、效果的展示等。上传和下载不用多说，需要注意的是不仅仅可以上传和下载本地，还支持URL连接即可，生效的训练，评估就需要详细的说明，还有就是数据集的制作。最需要说明的就是训练和评估，尤其需要注意的是，不是在本系统进行训练和评估，是管理，是调用，是触发。也就是说需要有能管理服务器的功能，才能进行这些操作，这是一个比较隐性的需求。
7. python的代码需要有详细的中文注释，包括函数、类、参数、返回值等。主要是业务含义，不能只写一些简单的参数说明。
